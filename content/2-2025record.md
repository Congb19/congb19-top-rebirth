## 重置 congb19-top 主站并升级

使用 Nuxt 全家桶开发，Github Pages 静态部署，属于 SSG 模式。

- 最初使用当时最新版本 **Nuxt 3 + Nuxt UI 2**，手搓了一下页面布局、黑暗模式能力
- 迁移了旧版主站的一些组件能力，使用 **VueUse** 进行了简化：电子木鱼、系统信息检测工具
- 增加了一些新的工具组件：键盘检测工具、屏幕坏点检测工具、反应速度测试工具
- 增加了一些动画效果：鼠标拖尾、单点元素动画、打字机效果，首页均可看到
- 升级至 **Nuxt 4 全家桶**：在全都按照官方指引升级到 v4 之后，发现此前手搓的主站布局、header 的布局和 Nuxt UI 默认的 Page 系列组件非常接近，于是再次简化，整体布局风格直接跟 Nuxt UI 完全一致

> **遗留问题 ✅ 已解决**
>
> 旧方案使用 Github Actions 进行 generate 再部署有问题，需要本地 generate 后把产物全部推送至代码仓。
>
> 现在的方案：**在 `.github/workflows/deploy.yml` 中完成打包和部署**，workflow 监听 `release` 分支，发布时只需提交 PR 将 `master` 合并至 `release`，合入后自动触发 action 进行部署，`.gitignore` 已将打包产物（`.output`、`dist`、`github_pages` 等）排除。
